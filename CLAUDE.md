# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What is elmq?

A Rust CLI for querying and editing Elm files â€” like jq for Elm. Uses tree-sitter-elm for parsing. Supports reading (`list`, `get`), writing with a `<verb> <scope>` convention (`set decl`, `patch`, `rm decl`, `add import`/`rm import`, `expose`/`unexpose`, `add variant`/`rm variant`, plus sub-declaration writers: `set let`, `set case`, `rm let`, `rm case`, `rm arg`, `rename let`, `rename arg`, `add arg`), project-wide operations (`mv` â€” rename a module and update all references; `refs` â€” find all references to a module or declaration; `rename decl` â€” rename a declaration project-wide; `move-decl` â€” move declarations between modules with import-aware body rewriting; `variant cases` â€” read-only companion that lists every case expression on a type with its enclosing function body and a stable per-site key, enabling `add variant --fill <key>=<branch>` to synthesize real branch bodies in the same call instead of leaving `Debug.todo` stubs; the top-level `refs` command also auto-routes constructor names through a classifier that categorizes every project-wide reference by syntactic role (case branch, expression position, function/lambda/let-binding refutable patterns), which is the same data `rm variant` surfaces as an advisory `references_not_rewritten` section alongside its mutation), and discovery (`grep` â€” regex search over Elm sources annotated with the enclosing top-level declaration; honors `.gitignore` and walks up for `elm.json` or falls back to CWD when none is found). All write commands confirm success with `ok` output (except `rm variant`, which emits its advisory `references_not_rewritten` section when applicable). Most subcommands accept multiple positional arguments per call (e.g. `list FILE...`, `get/rm decl FILE NAME...`, `refs FILE [NAME...]`, `add/rm import FILE ARG...`, `expose/unexpose FILE ITEM...`, `move-decl FILE --to TARGET NAME...`); multi-arg output is framed as `## <arg>` blocks in input order. `get` additionally supports multi-file reads via `-f`: `get -f FILE NAME... [-f FILE NAME... ...]` reads declarations across several files in one call, framing output as `## Module.decl` (or `## file:decl` without `elm.json`). Write commands accepting inline content use `--content` (`set decl`) or `--body` (`set let`, `set case`) flags as stdin alternatives (exactly-one-of). A mismatch between a target's name and `--name` on `set decl` or `set let` is now an error; use `rename decl` or `rename let` instead. Sub-declaration commands with `--line` flags use absolute file line numbers matching `get` output.

## Build & Test Commands

```sh
cargo build --release --locked   # compile release binary
cargo test                       # run all tests
cargo clippy -- -D warnings      # lint (must be warning-free)
cargo fmt --check                # check formatting
cargo fmt                        # auto-format
cargo install --path .           # install locally
```

To run a single test: `cargo test <name>`

Rust toolchain is pinned in `rust-toolchain.toml` (rustup installs it automatically).

## Architecture

- **`src/lib.rs`** â€” Public types (`FileSummary`, `Declaration`, `DeclarationKind`). The library crate root.
- **`src/parser.rs`** â€” All tree-sitter-elm parsing logic: `parse()` returns a tree, `extract_summary()` walks it to produce a `FileSummary`, `find_declaration()` looks up by name, `extract_declaration_name()` parses a name from source text. Also exposes `first_error_location()` (walks the CST to find the first ERROR/MISSING node, returns 1-indexed `line:col`) and `ensure_clean_parse()` (write-path precondition â€” parses source and bails with a `refusing to edit <path>: file has pre-existing parse errors` error if the tree has any error nodes, so write commands never splice edits into a damaged CST). Unit tests live here inline.
- **`src/imports.rs`** â€” Import context abstraction: `ImportContext` models how a file imports other modules. Provides `resolve_prefix()`, `resolve_bare()`, `emit_ref()`, `ensure_import()`, and `render_imports()`. Handles Elm auto-imported modules. Used by `refs.rs` and `move_decl.rs`.
- **`src/project.rs`** â€” Project discovery: find `elm.json`, parse `source-directories`, resolve module names from file paths, enumerate all `.elm` files. Used by `mv`, `refs`, `rename`, and `move-decl`.
- **`src/writer.rs`** â€” All write operations: `upsert_declaration()`, `patch_declaration()`, `remove_declaration()`, `add_import()`, `remove_import()`, `expose()`, `unexpose()`, `rename_module_references()`, `rename_module_declaration()`, plus `atomic_write()` for safe file writes. **Write choke point: `validated_write(path, content, op)`** â€” re-parses the output buffer with tree-sitter-elm and rejects with `rejected '<op>' write to <path>: output would not parse at <line>:<col>` if the result has ERROR/MISSING nodes, then delegates to `atomic_write`. Every elmq command that mutates Elm source routes its final buffer through `validated_write` (not `atomic_write` directly) so syntactically broken files never reach disk, whether from user-supplied garbage or an internal splicing bug.
- **`src/refs.rs`** â€” Project-wide reference finding: `find_refs()` locates all references to a module or declaration across the project, resolving qualified, aliased, and explicitly-exposed references via tree-sitter. Delegates to `ImportContext` internally.
- **`src/move_decl.rs`** â€” Move-declaration orchestration: `execute_move_declaration()` moves declarations between modules with import-aware body rewriting, automatic helper detection, dependency analysis, and project-wide reference updates. Moved declarations are exposed in the target only when referenced by other project files or the source; if the source still references moved names, an import for the target is added automatically.
- **`src/variant.rs`** â€” Variant command: `execute_add_variant()`, `execute_rm_variant()`, `execute_cases()`, `execute_refs_for_constructor()`, and `resolve_constructor_in_file()`. `add`/`rm` add/remove constructors from custom types and propagate through all case expressions project-wide. `cases` is a read-only walker that emits every case expression on the target type with its enclosing function body and a stable site key. `execute_refs_for_constructor` is a read-only walker that classifies every project-wide reference to a specific constructor into `case-branch`, `case-wildcard-covered`, `function-arg-pattern`, `lambda-arg-pattern`, `let-binding-pattern`, or `expression-position`; invoked both by the top-level `elmq refs` dispatcher (via `resolve_constructor_in_file` for name routing â€” probes whether a name is a constructor of a type declared in the target file, so `run_refs` can decide between the classifier path and today's decl-based `find_refs_batch` path) and by `rm variant` to populate its `references_not_rewritten` advisory field for every non-cleanly-removable reference. Constructor refs and `rm` share `collect_constructor_sites()`, a two-pass classifier that walks every `upper_case_qid` with an ancestor-field context (reset at expression-body boundaries, set by pattern-hosting fields) and a second targeted pass that detects `case_of_expr` nodes whose wildcard branch already covers the removed constructor. `add` accepts a `fills: HashMap<String, String>` parameter that replaces the default `Debug.todo "<VariantName>"` stub with the caller-supplied branch text at matching sites; unmatched fill keys fail validation before any file is touched. `rm` mutates as today (strip variant from the type, remove every cleanly-removable case branch including nested patterns like `Just Increment`) and is advisory-not-gating: non-clean references are reported alongside the mutation so the agent can fix them before running `elm make`. Builds a constructor map for type resolution, walks `case_of_expr` nodes, handles tuple patterns and nested `nullary_constructor_argument_pattern` wrappers. Site-collection helpers (`collect_case_sites_generic`, `compute_site_keys`) are defined in `analysis.rs` for sharing with sub-decl writers.
- **`src/guide.md`** â€” Agent integration guide, compiled into the binary via `include_str!()` and emitted by `elmq guide`.
- **`src/grep.rs`** â€” Grep command: `execute()` entry point with `GrepArgs` struct; scans Elm files for regex (or literal, via `-F`) matches and annotates each hit with its enclosing top-level declaration via a tree-sitter-built offsetâ†’decl map. Filters matches inside `--` / `{- -}` comments and string literals by default (opt back in with `--include-comments` / `--include-strings`). `--definitions` filters to matches at the declaration name site (byte-range coincidence with `[name_start, name_end)`). `--source` emits full declaration source blocks deduped by `(file, decl)`, framed as `## Module.decl` (single block stays bare). The two compose: `--definitions --source` is a one-call definition lookup. Project discovery walks up for `elm.json` honoring `source-directories`, or falls back to recursively walking the CWD when none is found; both paths honor `.gitignore` via the `ignore` crate. Emits compact `file:line:decl:text` or NDJSON. Exit codes match ripgrep (0/1/2).
- **`src/analysis.rs`** â€” Cross-cutting site-collection and site-key computation helpers used by variant and sub-decl commands: `collect_let_sites()`, `collect_case_sites_generic()`, `collect_binders_in_scope()`, `compute_site_keys()` (progressive-qualification algorithm for stable site keys across multiple case/let contexts).
- **`src/writer/let_binding.rs`** â€” Sub-declaration writers for let bindings: `upsert_let_binding()`, `remove_let_binding()`, `rename_let_binding()`.
- **`src/writer/case_branch.rs`** â€” Sub-declaration writers for case branches: `upsert_case_branch()`, `remove_case_branch()`.
- **`src/writer/function_arg.rs`** â€” Sub-declaration writers for function parameters: `insert_function_arg()`, `remove_function_arg()`, `rename_function_arg()`.
- **`src/cli.rs`** â€” clap derive definitions with new nested enum structure: `Cli`, `Command` (dispatches to verb-level enums `Set { command: SetCommand }`, `Rm { command: RmCommand }`, `Rename { command: RenameCommand }`, `Add { command: AddCommand }` alongside non-nested verbs like `Patch`, `Expose`, `Unexpose`, `Mv`, `MoveDecl`, `Refs`, `Grep`, `Guide`), `SetCommand`, `RmCommand`, `RenameCommand`, `AddCommand` sub-enums, `Format`.
- **`src/main.rs`** â€” Thin CLI entry point. Reads file, calls parser/writer, formats output. Dispatches through nested verb enums (`Set`, `Rm`, `Rename`, `Add`) to scope-specific handlers. The `mv`, `refs`, `rename decl`, `move-decl`, and `variant` commands use `project.rs` for multi-file operations. Not part of the library crate.

The library (`lib.rs` + `parser.rs` + `imports.rs` + `writer.rs` + `writer/let_binding.rs` + `writer/case_branch.rs` + `writer/function_arg.rs` + `analysis.rs` + `project.rs` + `refs.rs` + `move_decl.rs` + `variant.rs`) is fully testable without the CLI binary.

- **`tests/`** â€” Integration tests per command: `get.rs`, `set.rs`, `patch.rs`, `rm.rs`, `import.rs`, `expose.rs`, `mv.rs`, `refs.rs`, `rename.rs`, `move_decl.rs`, `variant.rs`, `grep.rs`, `guide.rs`.
- **`.claude-plugin/`** â€” Claude Code plugin that injects `elmq guide` output into sessions via a SessionStart hook. Activates conditionally when an Elm project (`elm.json`) is detected and `elmq` is on PATH. Install via `/plugin install elmq@caseyWebb`.
- **`benchmarks/`** â€” Dockerized benchmark harness measuring token usage on Elm coding tasks. Two arms: `control` (Claude without elmq guidance) and `treatment` (elmq CLI guidance from `elmq guide` delivered as `CLAUDE.md` in the treatment workdir, which propagates to spawned `Task`/`Agent` subagents). Answers Q1: does elmq save tokens given Claude knows how to use it? Run via `./benchmarks/run.sh [control|treatment] [-n N]` â€” launches parallel runs with scoped results directories.

## Conventions

- Requires Rust edition 2024 (nightly features like let-chains are used)
- Run `cargo fmt` before committing
- PR titles must follow [Conventional Commits](https://www.conventionalcommits.org/) (enforced by CI, used for squash merge commit messages)
- Test fixtures go in `test-fixtures/`; add/update sample `.elm` files when adding parser features
- `openspec/` contains spec-driven development artifacts (changes and specs)
